Status Zero

A Node.js automation tool built during my time at SelectQuote that eliminated manual status check-ins by scraping git commit logs from the previous workday and submitting them to Status Hero automatically. It was designed to run on a cron job so there was never any need to think about check-ins again.

Status Hero rebranded to Steady in 2024 and no longer works the same way, so this project is now defunct—but it was a fun one that seems worth sharing.

Stack

  • Node.js (ES6) — runtime and scripting environment
  • Puppeteer (v2) — headless browser automation for logging into Status Hero and submitting the check-in form
  • Moment.js — date manipulation for calculating the previous workday
  • js-yaml — parses the YAML config file for credentials and project directories
  • Jest — unit testing for business day logic

Architecture

The tool was intentionally minimal—three modules wired together in a single orchestration script.

Configuration

A config.yaml file held Status Hero credentials, the git author name to filter by, and a list of local repository paths to scan. Two control flags allowed opting out of the actual form submission (willSubmit) or running the browser in non-headless mode (willRunHeadless) for debugging.

Business Day Calculation

calendar.js resolved the previous workday from today's date. If today was Monday it stepped back three days to Friday; otherwise it stepped back one day.

Git Log Collection

fetchAllGitLogs.js iterated over the configured repository directories and ran a git log command in each, filtered to the previous workday and the configured author. Commit subjects and bodies were collected using the %s%n%b format and concatenated into a single string ready for the Status Hero form field.

Browser Automation

The main entry point launched a Puppeteer browser, navigated to the Status Hero sign-in page, typed credentials, and submitted the login form. It then opened the status edit page, cleared the existing input, pasted the aggregated commit log, checked the goals-completed checkbox, and submitted—all conditionally gated on the willSubmit flag so a dry run was always one config change away.

Cron Integration

The script was stateless and side-effect-free outside of the browser session, making it safe to schedule with a standard cron entry. A typical setup ran it each weekday morning before standup.
